## 🔐 Sécurité - Décomposition

### ✅ BIEN FAIT:
- **CSRF**: Tokens 256-bit, hash_equals() timing-safe, validation avant mutations
- **SQL**: PDO prepared statements PARTOUT (50+ verificati)
- **Sessions**: HTTPOnly (XSS safe), SameSite (CSRF safe), use_only_cookies
- **Password**: bcrypt with password_verify(), session_regenerate_id()
- **Transactions**: commit/rollback pour cohérence stock
- **Error handling**: try/catch PDOException, pages erreur élégantes

### ❌ À CORRIGER:
- **Données sensibles**: nom_titulaire, numero_carte, CVV → pseudo-validées
- **Authentication**: Hash visible source code, pas multi-user
- **File upload**: MIME type validation côté client (faille!)
- **XSS**: htmlspecialchars parfois oublié (chemin image, données DB)
- **Rate limiting**: AJAX peut être spammée (100+ articles possible)

## 📋 6 CORRECTIONS PRIORITAIRES

### CRITIQUE #1: Validation Paiement (1h)
**Problème**: `$_POST['numero_carte']` pas validé → injection possible  
**Fix**: Regex 16 chiffres, Luhn algorithm, format MM/YY pour expiration  
**Fichier**: `traitement_paiement.php` ligne 10  
**Impact**: Sécurise données critiques

### CRITIQUE #2: Hash Admin (2h)
**Problème**: `$hash_sauvegarde = '$2y$10$...'` visible source code PHP  
**Fix**: Migrer hash en table `admin_users` + `password_hash` colonne  
**Fichiers**: `login.php`, créer migration SQL  
**Impact**: Production-ready, sécurité++

### CRITIQUE #3: XSS - htmlspecialchars (2h)
**Problème**: `echo $path` sans escape, `$_GET['cat']` utilisé avant validation  
**Fix**: Scaper TOUTES les sorties HTML, valider $_GET contre DB  
**Fichiers**: `index.php`, `catalogue.php`, `gestion.php`  
**Impact**: Prévient code injection client

### CRITIQUE #4: File Upload (1.5h)
**Problème**: Validation MIME côté client = faille (attacker peut mentir)  
**Fix**: `fileinfo_file()` vraie MIME, magic bytes check, `realpath()` validate  
**Fichiers**: `admin.php`, `modifier.php` ligne 55  
**Impact**: Shell upload prevention

### CRITIQUE #5: Rate Limiting AJAX (1h)
**Problème**: `ajouter_panier.php` peut être spammé (1000 articles possible)  
**Fix**: Throttle JavaScript + limite serveur (10 par item, 100 total)  
**Fichiers**: `script.js`, `ajouter_panier.php` ligne 20  
**Impact**: Abuse mitigation

### CRITIQUE #6: footer.php (0.5h)
**Problème**: `foret.php` l'inclut mais ... oh attends, il existe!  
**Status**: ✅ DÉJÀ CRÉÉ (vérifier contenu)
